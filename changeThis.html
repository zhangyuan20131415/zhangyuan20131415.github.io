关于call、apply、bind的区别

我们经常遇见的call,apply,bind差别问题，今天来总结一下吧！

这三个方法的存在都是为了改变函数执行的上下文,既改变this的指向。call和apply会在改变this指向之后直接执行，但是bind是在改变上下文之后返回一个函数，等到我们需要执行的之后再调用该函数即可。

关于call

call方法接收的第一个参数是要改成的执行上下文的对象，后续接收的参数都作为调用一个函数的参数。

				
                    Math.min.call(null, 1, 2, 3]) // 1

关于apply

apply接收两个参数，第一个是要改变的执行上下文的对象，第二个是要调用的函数的参数。

                    
                        Math.min.apply(undefined, [1, 2, 3]) // 1

关于bind

bind()方法创建一个新的函数，在bind()被调用时，这个新函数的this被bind的第一个参数指定，其余的参数将作为新函数的参数供调用时使用。 

                    
                       var binder = Math.min.bind(undefined, 1, 2, 3)
                       binder() //1

关于用call或者apply实现继承

                    
                       function People(name) {
                           this.name = name
                           this.sayHi = function() {
                               console.log('hi, i\'m ${this.name}')
                           }
                       }
                       function Person(name) {
                           People.call(this, name) || People.apply(this, [name])
                       }
                       var person = new Person('lili')
                       person.sayHi() // 'lili'
